Core Concepts of Convolutional Neural Networks (CNNs)

Understanding the Architecture of Visual and Spatial Data Processing

The Three Foundational Inductive Biases

  • Locality: Processing data in small, distinct patches.
  • Parameter Sharing: Applying the exact same rules everywhere.
  • Translation Equivariance: Recognizing patterns regardless of their spatial position.

1. Locality & 2. Parameter Sharing

  • Local Receptive Fields: Each filter looks at a small patch (e.g., 3x3) of the input — not the entire image.
  • Weight Sharing: The exact same filter is slid across the entire input, dramatically reducing parameters.

3. Translation Invariance & 4. Hierarchical Learning

  • Translation Invariance: Because the same filter is applied everywhere, a pattern detected in the top-left will also be detected in the bottom-right. The network generalizes spatial positions.
  • Compositional Representation:
    • Early layers: Learn low-level features (edges, corners).
    • Middle layers: Learn mid-level patterns (textures, shapes).
    • Deep layers: Learn high-level semantics (faces, complete objects).

5. Pooling (Spatial Subsampling) & 6. Depth

  • Pooling: Operations like Max or Average pooling reduce spatial dimensions.
    • Provides spatial robustness (invariance to small distortions).
    • Reduces computational load.
    • Widens the receptive field in deeper layers.
  • Depth / Multi-channel Processing: Filters operate across all input channels (e.g., RGB) simultaneously, allowing the network to combine rich information.
