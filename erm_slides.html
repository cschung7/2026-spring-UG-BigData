→ 반복 패턴이 있는 쿼리는 추가 비용 없이 더 정확하게 검색
문제 1
Stateless 반복 비용

Query expansion이나 iterative retrieval은 매번 새로 계산. 같은 쿼리 패턴이 반복돼도 누적 학습 없음. inference cost가 linearly 발생.

문제 2
Index-side Drift

Key expansion은 오프라인 전처리에 의존. Heuristic 업데이트는 downstream utility와 정렬 약함 → semantic drift + 노이즈 누적.

핵심 통찰
Zipf 분포 활용

실세계 쿼리는 Zipf 분포를 따름 — 소수의 문서에 트래픽 집중. 모든 key를 업데이트할 필요 없이 선택적 lazy update로 충분.

02 METHODOLOGY

세 가지 패러다임 비교

Paradigm A
Query Expansion (QE)
  • 쿼리를 inference-time에 확장
  • 문서 공간에 정렬시킴
  • 단점: 매 query 후 버려짐
  • 누적 학습 불가
Paradigm B
Key Expansion (KE)
  • 문서 key를 쿼리 공간에 맞춰 오프라인 보강
  • Persistence는 있음
  • 단점: 높은 전처리 비용
  • Semantic drift, 노이즈 누적
Paradigm C — ERM
Evolving Retrieval Memory
  • QE의 순간적 이득을 index에 영구 반영
  • 정답 검증된 신호만 선택 반영
  • Zero inference-time overhead
  • 수렴 이론 증명됨
이론적 등가성 (Key theorem): 논문은 표준 similarity 함수 하에서 query expansion ≡ key expansion임을 수학적으로 증명. 즉 ERM은 optimal query expansion의 이득을 index에 amortize할 수 있음이 보장됨.
sim(q_expanded, k) = sim(q, k_expanded)   // under standard inner-product similarity

🏦 경제학 analogie — 중앙은행 정책 학습
Correctness Gate는 중앙은행이 "이 금리 결정이 인플레이션 목표에 맞았는가?"를 사후 검증하는 것.

Atomic Attribution은 특정 지표(실업률, CPI)가 그 결과에 얼마나 기여했는지 분리 귀속.

Norm-Bounded Update는 급격한 정책 변화 없이 안정적으로 모델을 업데이트하는 점진적 조정.
# ERM pseudo-logic
for query in incoming_queries:
  result = rag_pipeline(query)
  if correctness_gate(result):
    signals = expand(query)
    for key in attributed(signals):
      key += bounded_update(signal)
      # ‖Δkey‖ ≤ ε (수렴 보장)

05 THEORETICAL FOUNDATION

왜 수학적으로 작동하는가?

Theorem 1 — 등가성
QE ≡ KE

표준 inner-product similarity 하에서 query를 확장하는 것과 key를 확장하는 것은 수학적으로 동치. 이는 ERM이 query-time 이득을 index에 amortize할 수 있는 이론적 근거.

Theorem 2 — 수렴
Bounded Update Convergence

Selective + norm-bounded 업데이트는 수렴이 증명됨. Key가 무한히 drift하지 않고 최적 상태로 안정적으로 수렴. Catastrophic forgetting 없음.

📊 경제학 analogie — 오차수정모형 (ECM)
ERM의 수렴 증명은 계량경제학의 오차수정모형(Error Correction Model)과 구조적으로 유사합니다.

ECM에서 장기 균형에서 이탈하면 단기 조정이 발생해 균형으로 복귀하듯, ERM의 norm-bounded update는 현재 key 상태에서의 편차를 제한하면서 점진적으로 optimal retrieval 상태로 수렴합니다.

수식: ‖key_{t+1} - key_t‖ ≤ ε  → Lyapunov stability 조건 만족
Δkey = α · P(correct | query, key) · attribution(signal → key)   // Correctness-weighted selective update

07 APPLICATIONS

경제·금융 분야 적용 시나리오

Scenario A
연구 리포트 인텔리전스

증권사 리서치 리포트 DB에 ERM 적용. 애널리스트가 반복적으로 조회하는 섹터/기업 쿼리가 쌓일수록 관련 문서 키가 자동 강화. NBER 논문 검색 시스템에 직접 적용 가능.

Scenario B
금융 Q&A 어시스턴트

"PER이 낮은 반도체 섹터 기업 찾아줘" 같은 복잡한 멀티홉 쿼리. 첫 응답이 정확할수록 관련 인덱스 키가 강화되어 유사 쿼리 반응속도 향상.

Scenario C
규제·컴플라이언스 검색

법규, 공시, 규제 문서 베이스. 컴플라이언스 담당자의 반복 질의 패턴이 index에 축적되어 희귀하지만 중요한 조항도 정확하게 검색.

🧠 학습 포인트 — RAG vs Fine-tuning vs ERM의 포지셔닝
Fine-tuning
모델 파라미터 수정. 대용량 도메인 데이터 필요. 비용 높음. 재학습 주기 느림.
Standard RAG
정적 인덱스. 빠른 배포. 하지만 쿼리에서 배우지 않음. 반복 쿼리에도 동일 비용.
ERM
인덱스가 쿼리로부터 학습. 추가 학습 비용 없음. 시간이 지날수록 성능 향상.
08 LIMITATIONS & FUTURE WORK

한계와 열린 연구 방향

Limitation 1
Correctness 판단 의존성

정답/오답을 판단하는 verifier의 품질에 전체 시스템이 의존. 검증이 어려운 오픈엔드 질문(예: "최적 금리는?")에는 gate 설계가 복잡해짐.

Limitation 2
Cold Start 문제

초기 쿼리가 적을 때는 일반 RAG와 동일. Zipf 분포의 이점이 실현되려면 충분한 쿼리 누적이 필요. 트래픽이 적은 도메인에서는 이득이 제한적.

Future Direction 1
Multi-modal ERM

현재는 텍스트 전용. 금융 차트, 테이블, 이미지가 포함된 리서치 리포트에 적용하려면 멀티모달 key 업데이트 메커니즘 필요.

Future Direction 2
Federated ERM

여러 기관이 쿼리 데이터를 공유하지 않고도 인덱스 경험을 federated 방식으로 축적. 금융기관 간 privacy-preserving 협력 학습 가능성.

비판적 시각: ERM의 이론적 수렴 보장은 convex 조건 가정. 실제 고차원 임베딩 공간에서의 non-convexity, 분포 시프트, adversarial 쿼리에 대한 robustness는 추가 연구 필요.

🎓 Take-away for 경제학 전공자
ERM은 베이지안 업데이트의 AI 버전으로 이해할 수 있습니다. 사전 확률(기존 index key) + 데이터 likelihood(검증된 쿼리 신호) → 사후 확률(업데이트된 key). 단, 업데이트가 norm-bounded이므로 과잉반응(overreaction) 없는 합리적 기대 형성에 가깝습니다.    |    arXiv: 2602.05152  ·  Hu et al., Feb 2026
