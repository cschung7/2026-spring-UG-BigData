방식 01 04 / 10
🔨 전통적 코딩
개발자가 모든 코드를 직접 작성. 완전한 이해, 완전한 노력.
Python 백엔드 — data_fetch.py
# data_fetch.py — 전통적 방식 import requests, json from datetime import datetime TICKERS = ["SPY", "QQQ", "GLD", "BTC-USD"] API_KEY = "YOUR_ALPHA_VANTAGE_KEY" def fetch_price(symbol): url = ( f"https://www.alphavantage.co/query?" f"function=GLOBAL_QUOTE&symbol={symbol}" f"&apikey={API_KEY}" ) r = requests.get(url) data = r.json() return { "price": data["Global Quote"]["05. price"], "change": data["Global Quote"]["10. change percent"], } def build_dashboard(): results = {} for t in TICKERS: results[t] = fetch_price(t) return results
개발 과정

① 아키텍처 설계 (~2시간)

데이터 흐름 계획: API → Python → JSON → HTML. 라이브러리 선택, API 문서 읽기.

② 모든 모듈 작성 (~6시간)

data_fetch.py, news_scraper.py, render_html.py, style.css — 모두 처음부터 작성.

③ 디버깅 & 테스트 (~4시간)

API 요청 제한, JSON 키 오류, CORS 문제, CSS 반응형 버그 등.

④ 배포 (~1시간)

GitHub Actions 설정, Pages 배포, 환경변수 등록.

⏱ 예상 총 소요 시간: 12~16시간

Python, REST API, HTML/CSS, 배포 파이프라인 전반에 걸친 숙련도 필요

바이브 코딩의 "바이브"란?

원하는 느낌과 동작을 설명하면, 모델이 구현 세부사항을 채워줍니다. 개발자의 역할이 직접 쓰기에서 방향 제시로 바뀝니다.

⏱ 예상 소요 시간: 1~3시간 (대부분 프롬프트 반복 + 검토)
